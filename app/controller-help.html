LRBridge Web Controller Help

Local browser control surface for Adobe Lightroom Classic through LRBridge.

What the Web Controller is for

LRBridge Web Controller is a local browser control panel for Lightroom Classic. It can move Develop sliders left or right, reset one slider, and run supported Lightroom actions.

It works like a simple button surface for Lightroom: press a button, LRBridge sends a command, Lightroom responds.

Feedback is not implemented yet

LRBridge sends commands, but it does not read Lightroom values yet

The Web Controller does not know the current Exposure, Contrast, Tint, Temperature, or other slider values inside Lightroom.

For now, LRBridge only sends commands. It does not receive live slider values back from Lightroom yet.

If you press +1, LRBridge tells Lightroom to move that slider one small step to the right. It does not know what the final Lightroom value became.

Safe everyday commands

What you want Command Meaning
Move slider right /adjust?slider=Exposure&amount=1 Positive amount moves the Lightroom slider to the right.
Move slider left /adjust?slider=Exposure&amount=-1 Negative amount moves the Lightroom slider to the left.
Reset one slider /reset?slider=Exposure Returns one Lightroom slider to its default value.
Run Lightroom action /action?action=setAutoTone Runs a supported Lightroom action through LRBridge.

Bitfocus Companion, Stream Deck, Loupedeck, and other control surfaces

The built-in Web Controller is not the only way to use LRBridge

LRBridge can also be controlled by any app, device, script, or hardware controller that can send HTTP requests.

That includes Bitfocus Companion, Stream Deck HTTP/request plugins, PowerShell, curl, scripts, browser shortcuts, and other HTTP-capable tools.

Loupedeck hardware was one of the reasons LRBridge was created. Loupedeck devices have excellent knobs for photo editing, but Loupedeck support is fading. Good hardware should not become useless just because the original software is no longer moving in the right direction.

LRBridge helps keep that hardware alive.

Bitfocus Companion is the main target for this workflow. With Companion and the Generic: HTTP Requests plugin/add-on, LRBridge commands can be mapped to Companion buttons, pages, and supported hardware devices.

The detailed Companion setup and copy-ready commands are on the Builder page.

Open Bitfocus Companion HTTP Builder

The Builder is mainly for Bitfocus Companion Generic HTTP setup. But other HTTP-capable tools can use the same commands too.

We are also working on a native Bitfocus Companion plugin. That should let users choose LRBridge sliders and actions from dropdown menus instead of copying commands manually.

PowerShell example

PowerShell can send a command directly to Lightroom through LRBridge: 

curl.exe "http://127.0.0.1:17891/adjust?slider=Exposure&amount=1"

Advanced dangerous commands

Reset group and Reset all are use-at-your-own-risk commands

/reset-group and /reset-all are not available in the Web Controller UI.

LRBridge still supports these endpoints for advanced users who want to call them manually from PowerShell, scripts, Bitfocus Companion, or another HTTP-capable tool.

They can send many commands to Lightroom very quickly. This can overload Lightroom and may freeze or crash Lightroom. Use them at your own risk.

Advanced command Example
Reset group /reset-group?group=Basic
Reset all known sliders /reset-all

PowerShell examples: 

curl.exe "http://127.0.0.1:17891/reset-group?group=Basic"
curl.exe "http://127.0.0.1:17891/reset-all"
