LRBridge Web Controller Help

Local browser control surface for Adobe Lightroom Classic through LRBridge.

What the Web Controller is for

LRBridge Web Controller is a local browser control panel for Lightroom Classic. It can move Develop sliders left or right, reset an individual slider, run supported Lightroom actions, and show Lightroom feedback values for visible sliders.

The Web Controller can run on the same PC as Lightroom, or from another browser device on the local network. That means a tablet, touchscreen monitor, old phone, laptop, or another computer can become a simple Lightroom control surface.

The Web Controller started as a practical side tool for testing LRBridge without configuring external hardware. It became useful on its own, especially for touchscreens, tablets, phones, secondary monitors, laptops, and other browser-capable devices.

The original idea behind LRBridge

LRBridge started from a real problem: Loupedeck Live had exactly the kind of knobs Lightroom Classic needs, but the official software did not give that hardware the experience it deserved.

After Logitech acquired Loupedeck, relying on the official software layer became the weak point. Bitfocus Companion gave the device a second life outside the original software stack.

Bitfocus Companion is a control-surface and automation application used to turn devices like Stream Deck, Loupedeck, Razer Stream Controller, and other supported hardware into customizable buttons, knobs, pages, and triggers. It is widely used in AV, broadcast, production, streaming, and custom control setups.

That solved the hardware side of the problem, but not direct Lightroom slider control. Companion can trigger shortcuts and send commands through supported integrations, but it does not have access to the Lightroom Classic SDK or the Develop slider API.

Other Lightroom control solutions already exist. MIDI2LR is solid for traditional MIDI controller setups, and LrSuperKeys is excellent for keyboard and mouse-wheel based control. Both are worth recommending for their intended use.

The problem is that neither approach is a clean fit for this Companion-first setup.

That is where LRBridge comes in. Companion sends simple HTTP commands to LRBridge. The LRBridge Lightroom plugin receives those commands from inside Lightroom Classic and applies the requested slider change, reset, or action through the SDK.

The result is a direct local control path for Lightroom Classic that works with Companion, browsers, phones, tablets, touchscreens, PowerShell, curl, scripts, HTTP request tools, and other button-deck-style devices.

Lightroom feedback in the Web Controller

LRBridge can show current Lightroom slider values

The Web Controller can show feedback values below supported Lightroom sliders. Feedback works for slider changes made from the Web Controller and for slider changes made directly inside Lightroom.

To keep Lightroom responsive, LRBridge requests feedback only for sliders that are currently visible in the Web Controller, and it sends changed values back to the browser.

Feedback is polling-based, not true native realtime feedback. A small delay is normal while moving sliders quickly.

Safe everyday commands

What you want Command Meaning
Move slider right /adjust?slider=Exposure&amount=1 Positive amount moves the Lightroom slider to the right.
Move slider left /adjust?slider=Exposure&amount=-1 Negative amount moves the Lightroom slider to the left.
Reset an individual slider /reset?slider=Exposure Returns one Lightroom slider to its default value.
Run Lightroom action /action?action=setAutoTone Runs a supported Lightroom action through LRBridge.

Bitfocus Companion integration methods

Use LRBridge with Companion in one of two ways

LRBridge can be controlled from Bitfocus Companion through Companion's built-in HTTP request tools, or through the native LRBridge Companion module.

In both cases, the command path is the same: Companion sends an HTTP command to LRBridge, LRBridge places it into its local command queue, and the Lightroom plugin applies the requested slider change, reset, or action inside Lightroom Classic. 

Loupedeck device -> Bitfocus Companion -> LRBridge app -> LRBridge Lightroom plugin -> Lightroom Classic

Method 1: Companion Generic HTTP Requests

Use Companion's built-in HTTP request tools to map LRBridge commands to Companion buttons, pages, and supported hardware controllers.

This method works with tools already integrated in Companion and does not require the native LRBridge Companion module. The Builder page provides copy-ready commands for slider adjustment, slider reset, and Lightroom actions.

Open Bitfocus Companion HTTP Builder

Method 2: Native LRBridge Companion module

Use the native LRBridge Companion module for a cleaner Companion setup. This lets you choose LRBridge sliders and actions from Companion dropdown menus, instead of manually copying HTTP paths.

Companion module GitHub repo:
https://github.com/ninoleto/companion-module-ninoleto-lrbridge

If the module is not available in the official Companion build yet, use the GitHub repo for the current development version and setup instructions.

Beyond Companion setup, the HTTP Builder is also useful for testing, scripting, curl, PowerShell, browser shortcuts, Stream Deck HTTP/request plugins, and any other tool that can send HTTP requests.

PowerShell example

PowerShell can send a command directly to Lightroom through LRBridge: 

curl.exe "http://127.0.0.1:17891/adjust?slider=Exposure&amount=1"

Advanced dangerous commands

Reset group and Reset all are use-at-your-own-risk commands

/reset-group and /reset-all are not available in the Web Controller UI.

LRBridge still supports these endpoints for advanced users who want to call them manually from PowerShell, scripts, Bitfocus Companion, or another HTTP-capable tool.

They can send many commands to Lightroom very quickly. This can overload Lightroom and may freeze or crash Lightroom. Use them at your own risk.

Advanced command Example
Reset group /reset-group?group=Basic
Reset all known sliders /reset-all

PowerShell examples: 

curl.exe "http://127.0.0.1:17891/reset-group?group=Basic"
curl.exe "http://127.0.0.1:17891/reset-all"
