LRBridge Web Controller Help

Local browser control surface for Adobe Lightroom Classic through LRBridge.
Current limitation: the Web Controller sends commands to Lightroom Classic, but it does not yet show live Lightroom slider feedback. Watch Lightroom Classic for the real result. Feedback/readback will be added through the future LRBridge state API.

How it works

LRBridge runs a local HTTP server. The Web Controller is a browser page that sends commands to that local server. The Lightroom plugin polls LRBridge, receives the next command, and applies it inside Lightroom Classic. 

Web Controller → LRBridge App → LRBridge Lightroom Plugin → Lightroom Classic

Local addresses

Service Address Purpose
Web Controller http://127.0.0.1:17892/ Browser control surface.
LRBridge HTTP API http://127.0.0.1:17891/ Command API used by the Web Controller and future Companion plugin.
LRBridge WebSocket ws://127.0.0.1:17890/ Experimental command input. HTTP is the recommended path for now.

Basic use

  1. Start LRBridge.
  2. Open Lightroom Classic and select a photo.
  3. Open the Web Controller.
  4. Use slider buttons, drag strips, tool buttons, reset buttons, and actions.

Slider controls

Each slider row contains -5, -1, Reset, +1, +5, and a drag strip. Amounts are Lightroom increment steps, not absolute numeric values. 

Exposure +1  → /adjust?slider=Exposure&amount=1
Exposure -5  → /adjust?slider=Exposure&amount=-5
Reset        → /reset?slider=Exposure

Drag strips

Drag left or right on a strip to send repeated small adjustments. LRBridge coalesces repeated slider adjustments in the command queue, so dragging should feel smoother than sending every tiny movement as a separate command.

Switches

Some Lightroom boolean-style controls are currently handled as slider adjustments. For example, profile corrections and chromatic aberration controls use On/Off buttons that send positive or negative adjustment steps.

Actions and tools

Action buttons call Lightroom develop actions such as Auto Tone, Auto White Balance, tool selection, Reset Crop, and Reset Transform. 

/action?action=setAutoTone
/action?action=setAutoWhiteBalance
/action?action=selectCropTool
/action?action=resetCrop
/action?action=resetTransforms

HTTP API examples

Command Example
Adjust slider /adjust?slider=Exposure&amount=1
Reset slider /reset?slider=Exposure
Reset group /reset-group?group=Basic
Reset all known sliders /reset-all
Run action /action?action=setAutoTone
Raw API help /api/help from the Web Controller server or /help from the LRBridge API server.

Companion HTTP cheat sheet

For copy/paste Companion Generic HTTP button paths for every supported slider and action, open the project file: 

docs\COMPANION_HTTP_CHEATSHEET.md

Regenerate the sheet after slider/action changes with: 

npm run generate:cheatsheet

Companion direction

The future Bitfocus Companion module should use the same HTTP API as the Web Controller. First target: reliable actions and slider control through HTTP. Second target: feedback through a stable /state endpoint.

Troubleshooting

Problem Check
Buttons do nothing Make sure LRBridge is running and Lightroom Classic is open.
Commands are queued but Lightroom does not move Make sure the LRBridge Lightroom plugin polling is running.
Web Controller does not open Open http://127.0.0.1:17892/ manually.
Need server-only mode Use npm run server. Use npm start for the full app.
